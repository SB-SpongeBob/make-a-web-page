Lesson 1: The Basics of the Web and HTML

1. WWW

World Wide Web (WWW) is Collection of HTML(HyperText Markup language) documents. While almost all types of files can exist on the web, any type of file can be linked to HTML documents that are the basis for everything on the web.

2. Major Pieces

Your computer is running a Web browser, the Internet, and servers. Your browser makes requests via the Internet to servers. These requests are using a protocol called http and the servers respond with files that your browser displays. Some examples of browsers are Internet Explorer, Chrome, Firefox.

  • HTTP – the main protocol of the web
  • Servers – computers that host the files that make up the web
  • Internet – the world’s largest computer network
  • Browser – a program that runs on your computer to display files found on the web
3. HTML(HyperText Markup Language) is made up of:
  • Text content (What you see)
  • Markup (what it looks like)
  • References to other documents, e.g. images, videos
  • Links to others
4. Intro to HTML Tags

  • Tags, once opened, need to be closed!
  • HTML Attributes <Tag Attr="value">contents</tag>
5. Types of HTML tags
  • Bold tag - <b> </b>
  • Italics tag - <em> </em>
  • Anchor tag <a> - makes a link, <a href=”www.reddit.com” sung </a>
  • Image tag - <img src="URL" alt="text">
    Image tag is a void tag as there is no contents and therefore do not require a closing tag.
    Src = source alt = alternative name
  • Break tag - <br>
    Break tag is a void tag and do not need a closing tag. Can be used multiple times to give multiple line breaks. It ends the line and it is a lineline element (stays within the line), although it creates multiple lines as it just creates white space within the line pushing the rest to the next line.
  • Paragraph tag - <p> contents </p>
    It makes the invisible box (block element) around the contents.
  • "\n" that Steve writes is how programmers represent a "newline" character in common coding languages such as C, Java, Python, etc.
  • Span/Div - They are same but Span inline element Div is a block element.Their only function is to contain contents.
    - The <span> tag is used to group inline-elements in a document.
    - The <span> tag provides no visual change by itself.
    - The <span> tag provides a way to add a hook to a part of a text or a part of a document.

2. HTML, CSS and DOM
  • HTML - HyperText Markup Language - the standard markup language used to create web pages. (Structure of the web pages)

Lesson 3: Adding CSS Style and HTML Structure

1. CSS (Cascading Style Sheets)

1) CSS is code written to control the "style" of HTML elements

  • Sheets - You can have several CSS files or Sheets that will be referenced in HTML and all of them will be used to find out the final and visible style for your page
  • Cascading – It means the most specific rule applies

2) Search and Replace (avoid repetition) - You can also think about CSS as a search and replace tool: you identify a class or a tag of the element you want to find (or match, in CSS terminology), and then what you want to do with it, or what property values to replace with different ones.

3) Order Matters - It also matters where you define the rules and in what order they are applied. Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions:

  • the default style of a browser (different browsers have slightly different styles)
  • stylesheet in a separate file (this is what I will be mostly using)
  • stylesheet inside HTML (this can be done for small projects but is not ideal)
  • inline style in an element (this can also be done but should be avoided)

4) Specifics Matter - "Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

5) How does CSS "control" HTML - By selecting by tag or class

6) How do I include CSS Styling in my web page?

  • Write CSS in the <head> of your HTML
  • Link your HTML to a separate CSS file
  • Write your style inline with your HTML
2. The Box Model

  • Content - The content of the box, where text and images appear
  • Padding - Clears an area around the content. The padding is transparent
  • Border - A border that goes around the padding and content
  • Margin - Clears an area outside the border. The margin is transparent
3. CSS Box sizing property: Box-sizing: border box

By default, the width and height of an element is calculated like this:
width + padding + border = actual width of an element
height + padding + border = actual height of an element
This means: When you set the width/height of an element, the element often appear bigger than you have set (because the element's border and padding are added to the element's specified width/height). The box-sizing property solves this problem. It allows to include the padding and border in an element's total width and height. No matter how I tweak padding and box size, the box size that I see stays the same – remember the margin is not included. In order to display in all browsers, add browser specific prefixes.

8. Additional notes - Comments: Use it sparingly!

Comments are lines of text that the computer ignores. They are just there so that programmers can leave notes for other people (or themselves) to read.

  • In HTML, code comments begin with <!-- and end with -->. Everything in between is ignored.
  • In CSS, code comments begin with /* and end with */.
